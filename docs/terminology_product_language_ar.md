# دليل المصطلحات: Technical vs UX Language

آخر تحديث: 2026-04-09

## 1) الهدف
توحيد لغة المنتج في تجربة المستخدم لتتسق مع رؤية "الدعم/الرعاية/العافية"، مع الحفاظ على العقود التقنية الحالية دون كسر.

## 2) القاعدة الأساسية
لا نعيد تسمية:
- database models
- enums
- API routes
- internal DTO fields

في هذه المرحلة، نُغيّر لغة:
- UI copy
- Content labels
- onboarding prompts
- notifications text

## 3) المصطلحات التقنية الحالية (تبقى كما هي الآن)
- `PATIENT`
- `patients/me/*`
- `PatientProfile`
- `patient journey`

مرجع: backend schema/controllers + frontend route structure.

## 4) مصطلحات UX المعتمدة المقترحة
| Technical/Internal | UX/Product Preferred (Arabic) | ملاحظات |
|---|---|---|
| Patient | مستفيد | الخيار الافتراضي العام |
| Patient Journey | رحلتي / رحلتي العلاجية / رحلة الرعاية | يختار المنتج صيغة واحدة ثابتة في V1 |
| Therapist/Practitioner | المختص / مقدم الرعاية | حسب السياق |
| Session | جلسة | ثابت |
| Booking | حجز | ثابت |
| Matching | مساعد الاختيار / التوجيه للمختص المناسب | يفضّل لغة مبسطة |
| Assessment/Test | اختبار سريع / تقييم أولي | بدون نبرة تشخيصية ثقيلة |
| Support Ticket | طلب مساعدة | في واجهة المستخدم |
| Care Chat Request | طلب تواصل | واضح وبسيط |

## 5) Terms to Avoid في UX
- "مريض" كصياغة افتراضية شاملة.
- "تشخيص" في سياقات لا تحتوي تشخيصًا سريريًا فعليًا.
- لغة قاسية/مقلقة في رسائل الأخطاء أو empty states.

## 6) إرشادات Copy Migration
1. طبقة أولى: تحديث النصوص المرئية للمستخدم فقط.
2. طبقة ثانية: تحديث مفاتيح الترجمة تدريجيًا عند الحاجة (بدون كسر i18n بسرعة).
3. إبقاء mapping توثيقي واضح بين كل label UX وidentifier التقني.
4. أي نص قانوني/سريري يُراجع مع Product/Legal قبل تعميمه.

## 7) أمثلة تنفيذية
- بدل: `لوحة المريض`
  - استخدم: `لوحة رحلتي`
- بدل: `بيانات المريض`
  - استخدم: `بيانات الحساب`
- بدل: `اختبار مرضي`
  - استخدم: `اختبار أولي`

## 8) ملاحظات مهمة للفريق
- لا تغييرات breaking على backend enums/routes الآن.
- لا refactor أسماء modules فقط لأسباب لغوية.
- الأولوية: وضوح تجربة المستخدم + اتساق الرسائل.

## 9) خطة قصيرة معتمدة
1. اعتماد glossary رسمي (هذا الملف) من Product.
2. تطبيقه على Mobile MVP أولًا.
3. تنفيذ wave لاحقة على web copy بشكل incremental.
