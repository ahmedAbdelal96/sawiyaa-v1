# Support and Help

Support and help are reassurance surfaces.

They should reduce friction, not add it.

## Help center

The help center should answer common questions with clear sections and short answers.

It should feel:

- calm
- organized
- easy to scan
- not like a legal wall

## Patient support

Patient support should help with:

- booking questions
- payment questions
- session access issues
- cancellation questions
- refund questions
- chat or access issues

## UX goals

- Show the right support entry point quickly.
- Use plain language.
- Avoid technical explanations unless the user needs them.
- Offer escalation when the issue needs a human review.

## Public help vs patient support

- **Public help** is for visitors and pre-login users.
- **Patient support** is for signed-in patients with account-specific issues.

They should be related but not identical.
