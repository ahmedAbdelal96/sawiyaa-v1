Confirmed working as designed.

3. **Onboarding credential `fileUrl` display**: Line 995 displays `item.fileUrl` directly as a `credentialMeta` text. Is this intentional to show practitioners their uploaded credential URLs? Could be a security/UX concern if those URLs are sensitive. **AUDIT-107**.

4. **Application status `cleanReasonText`**: The regex on line 222 only strips "Reason|سبب الرفض" prefixes. What if the backend sends reasons prefixed differently? This may silently pass through unwanted prefixes.

## Findings to Register (AUDIT-105 onwards)

**AUDIT-105 — `formatRequirementLabel` used instead of i18n utility in dashboard and onboarding**
- Files: `fayed-mobile/app/(practitioner)/index.tsx` line 798-804; `fayed-mobile/app/(practitioner)/onboarding.tsx` line 529
- Evidence: `formatRequirementLabel(item)` title-cases raw requirement keys (e.g., "PRACTITIONER_OTP_VERIFIED" becomes "Practitioner Otp Verified") displayed as chip labels without Arabic translation.
- Impact: Practitioner sees English labels for missing requirements on the dashboard even when app is in Arabic.

**AUDIT-106 — `cleanReasonText` regex is fragile for rejection reason cleaning**
- File: `fayed-mobile/app/(practitioner)/application-status.tsx` line 222
- Evidence: `value.replace(/^(Reason|سبب الرفض)\s*:\s*/i, "").trim()` — only handles two prefix variants. Backend may send other formats.
- Impact: Rejection reason may display with prefix artifacts.

**AUDIT-107 — Credential `fileUrl` displayed as raw text in onboarding**
- File: `fayed-mobile/app/(practitioner)/onboarding.tsx` line 995
- Evidence: `<Text color={theme.colors.textSecondary} style={styles.credentialMeta}>{item.fileUrl}</Text>`
- Impact: Internal credential URLs exposed to practitioner in onboarding screen.

**AUDIT-108 — Hardcoded emoji "⏳" in onboarding checklist status text**
- File: `fayed-mobile/app/(practitioner)/onboarding.tsx` lines 873, 925
- Evidence: `<Text>⏳ {t("practitioner.onboarding.requiredDocuments.pendingReview")}</Text>`
- Impact: Emoji used alongside i18n string — not portable across all locales/fonts.

**AUDIT-109 — `I18nManager.isRTL` used instead of `getAppDirection(i18n.language)` in availability screen**
- File: `fayed-mobile/app/(practitioner)/availability/index.tsx` line 91
- Evidence: `const isRTL = I18nManager.isRTL` — reads device system RTL flag, not the app's language direction.
- Impact: Availability screen may display incorrectly when app language differs from device RTL setting.

**AUDIT-110 — Inline Arabic default values in availability validation messages**
- File: `fayed-mobile/app/(practitioner)/availability/index.tsx` lines 240-244, 268-272, 286-289, 296-299, 365-367, 370-373, 381-384, 390-394
- Evidence: Multiple uses of `t("key", { defaultValue: "Arabic text" })` pattern throughout the availability screen.
- Impact: These Arabic defaults suggest the i18n translation keys may not be fully populated for Arabic. If the translation system falls back to the default value, the app functions but the translation files are incomplete.

**AUDIT-111 — Hardcoded English error strings in messages thread screen**
- File: `fayed-mobile/app/(practitioner)/messages/[id].tsx` lines 17-20
- Evidence: `title="Conversation unavailable" message="We could not open this conversation."` — not i18n'd.
- Impact: Error state shown when `conversationId` is missing displays English text regardless of app locale.

**AUDIT-112 — Hardcoded Arabic strings in support conversation screen**
- File: `fayed-mobile/app/(practitioner)/support/[id].tsx` lines 76, 100 (header title), 121 (empty state), 152 (closed banner), 163 (composer placeholder)
- Evidence: `t("messages.inbox.sourceSupport", "محادثة الدعم")` — Arabic default inline. Closed banner has `"هذه المحادثة لا تقبل رسائل جديدة حاليًا."` hardcoded.
- Impact: Same pattern as AUDIT-110 — suggests i18n keys may not be fully registered in translation files.

**AUDIT-113 — Inline Arabic defaults in availability empty/placeholder states**
- File: `fayed-mobile/app/(practitioner)/availability/index.tsx` lines 725-729, 1027-1032
- Evidence: `t("practitioner.availability.selectedSlotsEmpty", { defaultValue: "لم تحدد أوقاتًا..." })`, `t("practitioner.availability.emptyExceptions", { defaultValue: "لا توجد استثناءات..." })`
- Impact: Same AUDIT-110 pattern — translation file gaps.
