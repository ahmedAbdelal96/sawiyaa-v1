SECTION A — CRITICAL BUGS (User-facing defects)
================================================================================

AUDIT-105 --------------------------------------------------------- CRITICAL
File: fayed-mobile/app/(patient)/sessions/success.tsx
Lines: 87-89
Issue: Raw backend session-status enums displayed directly to users as i18n keys
Evidence:
  {params.status === "PENDING_PAYMENT"
    ? t("patientSessionsFlow.statuses.PENDING_PAYMENT")
    : t("patientSessionsFlow.statuses.CONFIRMED")}
Severity: Users see raw enum strings "PENDING_PAYMENT" / "CONFIRMED" when the
  i18n keys are not found in the translation files. Payment confirmation is a
  critical post-payment state — users may be confused about their booking status.
Labels: [CRITICAL] [RAW-ENUM] [COPY] [i18n]

AUDIT-106 --------------------------------------------------------- CRITICAL
File: fayed-mobile/app/(patient)/support/new.tsx
Line: 109
Issue: Raw SupportTicketType enum values used as both i18n key and fallback
Evidence:
  {t(`support.categories.${category}`, category)}
  category values include: GENERAL, BOOKING, PAYMENT, SESSION, TECHNICAL,
  ACCOUNT, CHAT, OTHER
Severity: When the i18n key is missing, the fallback is the raw enum string
  (e.g., "BOOKING"). Users see backend enum values instead of human-readable
  category labels.
Labels: [CRITICAL] [RAW-ENUM] [COPY] [i18n]

AUDIT-107 --------------------------------------------------------- HIGH
File: fayed-mobile/app/(patient)/profile-notifications.tsx
Line: 162
Issue: formatNotificationType() produces humanized slugs WITHOUT i18n
Evidence:
  <Text weight="500">
    {formatNotificationType(item.typeSlug)}
  </Text>
  formatNotificationType (account-utils.ts:16-22) does:
    return typeSlug.split("_").map(part => part[0]+part.slice(1).toLowerCase()).join(" ");
  e.g. "SESSION_REMINDER" → "Session Reminder", "PAYMENT_CONFIRMED" → "Payment Confirmed"
Severity: Output is locale-agnostic (not AR-aware) and not sourced from the i18n
  system. In Arabic mode, notification type labels remain in English sentence-case.
  If a typeSlug has no underscores (e.g. "NEWS"), output is unchanged.
Labels: [HIGH] [COPY] [i18n] [RTL]

OBS-1: matching/questions.tsx:206 — HARDCODED PROGRESS
  const progress = 60;
  Progress indicator always shows 60% regardless of actual form completion.
  Not a user-facing string bug, but a UX inaccuracy. No i18n impact.
  Labels: [MINOR] [UX]

OBS-2: care-chat/new.tsx:307-313 — INLINE ERROR EXTRACTOR
  function extractApiErrorMessage(err: unknown): string {
    if (typeof err === "object" && err !== null && "message" in err) {
      const e = err as { message: string };
      return e.message;
    }
    return "An error occurred. Please try again.";
  }
  Fallback string is hardcoded English. Should use t() key for AR mode.
  Impact: Low — only fires on truly unknown error shape.
  Labels: [MINOR] [i18n]

OBS-3: profile.tsx:161 — ID PILL DISPLAY
  ID: P-{(user?.id ?? "0000").slice(0, 4).toUpperCase()}
  Displaying truncated user ID as a user-facing identifier. Not an enum display
  issue but may confuse users who expect a full ID or named identifier.
  Labels: [MINOR] [UX]
