FINDING 1 — CRITICAL: Raw presentationStatus Enum Leak in Session Chat Badge
================================================================================
File: fayed-frontend-v1/src/features/chat/components/SessionChatPanel.tsx
Line: 355
Evidence:
  {session?.presentationStatus.replaceAll("_", " ")}

The SessionChatPanel renders a badge showing the raw session.presentationStatus
string (e.g., "IN_PROGRESS", "JOINABLE", "NO_SHOW") with underscores replaced
by spaces but NEVER passed through i18n t(). The underscore replacement
produces English-readable labels ("IN PROGRESS") but these are hard-coded
English strings — no translation, no locale adaptation.

This is the AUDIT-053 regression: practitioner chat panel showing raw enum.

Impact: practitioner sees raw status labels in the chat panel status badge,
bypassing all i18n infrastructure. A practitioner with Arabic locale still
sees "IN PROGRESS" not "قيد التنفيذ".

Recommendation: Replace with t(`sessions.presentationStatus.${session.presentationStatus}`).

  function formatMoney(value: string, currency: string, locale: string) {
    const numeric = Number(value);
    if (Number.isNaN(numeric)) return `${value} ${currency}`;
    return new Intl.NumberFormat(locale === "ar" ? "ar-EG" : "en-US", {
      style: "currency",
      currency,
      minimumFractionDigits: 2,
      maximumFractionDigits: 2,
    }).format(numeric);
  }

displayed as raw numeric strings.

Verdict: PASS — currency formatting is correctly applied.
