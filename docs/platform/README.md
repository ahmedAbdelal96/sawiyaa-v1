# Fayed Platform Docs

This is the canonical English documentation set for the Fayed healthcare platform.

It explains the platform as it exists today:

- public web
- patient web
- practitioner web
- admin web
- backend service boundaries
- money, sessions, chat, support, and content flows
- the Fayed Clinical Warmth design language

Use this folder as the first stop when you need to understand how the product works or how the screens should feel.

## Source of truth

When a doc and the code disagree, the code wins.

This docs set should be updated whenever the actual product changes:

- a new route is added
- a role or permission changes
- a payment or refund flow changes
- a patient journey screen changes
- a chat state or cancellation policy changes
