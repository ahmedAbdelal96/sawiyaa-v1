# Components Boundary

- `ui/` for generic reusable primitives
- `shared/` for app-wide components used across multiple areas
- `auth/` for authentication-specific UI
- `public/` reserved for public website sections
- `patient/` reserved for patient-area UI
- `practitioner/` reserved for practitioner-area UI
- `admin/` reserved for admin and operations UI

Layout shells live in `src/layout/`.
